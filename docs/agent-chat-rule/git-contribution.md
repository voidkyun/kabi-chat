# Git操作に関するユーザーの発言の解釈ルール

- ユーザーが実装を求めたときはissue番号を先頭に付けた{n-branch-name}の形式のブランチにコミットしてプッシュし、PR作成まで行うものとする。
- 単に「レビューして」と言った場合には、現在のブランチを確認してgithub mcpを利用して下記のタスクを行うものとする。
  - ブランチに紐づくPRを特定
  - PRの内容をレビュー
  - PRに大きいmarkdownコメント1個を投稿
- ユーザーがレビュー後の修正を求めたときはプッシュしてPRに修正内容を一つの大きいmarkdownコメントで投稿するまで行うものとする。
